App6 Flight-Control Proof of Concept

This GitHub Pages site presents a real-time systems project built on an ESP32 using FreeRTOS. The project models a Lockheed Martin pilot-training flight-control support node, where timing matters just as much as correctness. The system separates sensor input, control logic, safety monitoring, rendering load, telemetry, heartbeat behavior, and an emergency override interrupt.

Project Summary

The potentiometer acts as the pilot yoke-position sensor, the pushbutton acts as an emergency override interrupt, the green LED is the system heartbeat, the red LED indicates a hard fault, and the blue LED shows override mode. The main real-time goal is to prove that the hard control path meets its deadlines while soft background tasks continue running.

What It Demonstrates

  • Periodic hard tasks for input, control, and safety
  • Soft tasks for rendering load, telemetry, and heartbeat
  • A minimal ISR that wakes a task using a binary semaphore
  • A one-slot queue that keeps only the newest control sample
  • Proof logging that shows latency and deadline status
